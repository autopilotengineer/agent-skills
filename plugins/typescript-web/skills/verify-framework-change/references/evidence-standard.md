# Evidence standard

Use sources in this order:

1. Official versioned API/configuration documentation.
2. Official migration or upgrade guide spanning installed to target versions.
3. Official release notes/changelog for intervening releases.
4. Official source and tests at the exact tag when documentation is ambiguous.
5. Authoritative runtime/package-manager documentation for peer constraints.

Record the URL, applicable version, claim supported, and access date when useful. A current unversioned page may describe only the latest release. Community sources can reveal questions to investigate but should not establish the final compatibility claim.
